# GitHub Labels and Tags Conventions

## Purpose
Provide a simple yet effective system for categorizing issues and pull requests on GitHub.

## Recommended Labels
- **bug:** For issues that indicate a defect. *(Color suggestion: red)*
- **enhancement:** For suggestions or improvements. *(Color suggestion: green)*
- **documentation:** For changes or additions to documentation. *(Color suggestion: blue)*
- **question:** For issues that are more exploratory or require clarification. *(Color suggestion: orange)*
- **help wanted:** For tasks needing additional assistance. *(Color suggestion: yellow)*
- **priority: high / medium / low:** Indicate the importance or urgency.

## Usage Guidelines
- **Consistency:** Always use these labels when applicable to streamline triage.
- **Review:** Regularly review and update labels to ensure they remain relevant.
- **Customization:** Projects may add additional labels, but the standard set should serve as the baseline.
