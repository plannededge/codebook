# GitHub Project and Directory Naming Guidelines

## Purpose

Establish a consistent naming convention across GitHub repositories, project directories, and IDE configurations to reduce ambiguity and improve discoverability.

## Guidelines

- **Repository Names:** Use lowercase letters and hyphens (e.g., `my-awesome-project`).
- **Directory Names:** Match repository names where possible. Use clear, descriptive names.
- **IDE Projects:** Configure project names to match the repository and directory names for consistency.
- **Branch Names:** Follow guidelines from [git/branching.md](../git/branching.md) with prefixes like `feature/`, `bugfix/`, `hotfix/`, etc.

## Examples

- Repository: `data-visualization-tool`
- Directory: `data-visualization-tool`
- Branch: `feature/add-chart-module`
